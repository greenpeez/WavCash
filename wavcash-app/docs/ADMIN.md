# Admin Dashboard
> Internal admin panel for managing data rights requests, payment distributions, and platform settings, accessible only via `admin.wav.cash`.

## Overview

The Admin Dashboard provides a multi-page interface for platform administration:

1. **Data Requests** — View and manage privacy/data rights requests (GDPR, CCPA, etc.) submitted via email
2. **Distributions** — Manual failsafe for distributing split payments by ISRC, with full activity timeline
3. **Settings** — Admin allowlist management

**Access:** `admin.wav.cash` only. The middleware redirects `admin.wav.cash/` to the data requests dashboard and blocks `/dashboard/admin/*` on any other hostname.

**Auth:** Privy authentication + email allowlist. Admin email is fetched directly from Privy (not the `users` table), so admins don't need a user-facing account. Only emails in the `admin_allowlist` table can access admin API routes. Non-admins see an "Access Denied" screen.

When a new data request arrives, all admins in the allowlist receive:
1. **In-app notification** — stored in `admin_notifications` table, shown in the bell dropdown
2. **Email alert** — sent from `WavCash Admin <privacy@noreply.wav.cash>` with request reference, sender info, and link to admin dashboard

## Email Flow

```
User emails privacy@wav.cash
  -> Porkbun MX (fwd1/fwd2.porkbun.com)
  -> Porkbun forwards to privacy@noreply.wav.cash
  -> Resend inbound MX on noreply.wav.cash
  -> Resend fires webhook -> POST /api/email/inbound
  -> Webhook verifies Svix signature, logs request to DB
  -> Sends acknowledgment email to sender
  -> Creates admin_notifications for all allowlisted admins
  -> Sends email alert to all allowlisted admins
```

### Acknowledgment Email (automatic)
- **From:** `WavCash Privacy <privacy@noreply.wav.cash>`
- **Subject:** "We received your data rights request — WavCash"
- **Content:** Confirms receipt, provides reference number (DR-XXXX), states 30-day completion window

### Completion Email (triggered from admin)
- **From:** `WavCash Privacy <privacy@noreply.wav.cash>`
- **Subject:** "Your data rights request has been completed — WavCash"
- **Content:** Summarizes what was done, includes supervisory authority links (CNPD, ICO, CPPA)

## Database

### `data_requests` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID (PK) | Auto-generated |
| `reference` | TEXT (unique) | Auto-generated `DR-0001`, `DR-0002`, etc. |
| `sender_email` | TEXT | From address of the incoming email |
| `sender_name` | TEXT | Extracted from email "From" field or manually set |
| `request_type` | TEXT | `access`, `deletion`, `correction`, `portability`, `opt-out`, `object`, `data_rights` |
| `status` | TEXT | `received` -> `in_progress` -> `completed` |
| `received_at` | TIMESTAMPTZ | When the email arrived |
| `completed_at` | TIMESTAMPTZ | When the completion email was sent |
| `completion_summary` | TEXT | Free text describing what was done |
| `original_subject` | TEXT | Subject line of incoming email |
| `original_body` | TEXT | Body text of incoming email |
| `resend_email_id` | TEXT | Resend's email ID for the inbound email |
| `created_at` | TIMESTAMPTZ | Row creation timestamp |

Reference numbers are auto-generated by a `BEFORE INSERT` trigger (`set_data_request_reference`).

### `admin_allowlist` table

| Column | Type | Description |
|--------|------|-------------|
| `email` | TEXT (PK) | Whitelisted admin email address |
| `created_at` | TIMESTAMPTZ | When the email was added |

Managed from the Settings page. Admins can add or remove emails. Self-removal is blocked. The primary admin (`ibukun.oyewumij@gmail.com`) is protected and cannot be removed by anyone — enforced both in the UI (no remove button) and server-side (DELETE handler rejects the request).

### `admin_notifications` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID (PK) | Auto-generated |
| `admin_email` | TEXT | Recipient admin email (not user_id — admins aren't in users table) |
| `title` | TEXT | Notification title |
| `body` | TEXT | Notification body |
| `read` | BOOLEAN | Whether the admin has seen it (default false) |
| `metadata` | JSONB | Additional data (reference, sender_email, etc.) |
| `created_at` | TIMESTAMPTZ | Creation timestamp |

Indexed on `(admin_email, read)`. RLS enabled with no policies — all access through service client.

## API Routes

### `POST /api/email/inbound`
Resend webhook endpoint. No auth required (verified via Svix webhook signature).

1. Verifies `svix-id`, `svix-timestamp`, `svix-signature` headers against `RESEND_WEBHOOK_SECRET`
2. Filters for `email.received` events to `privacy@noreply.wav.cash`
3. Fetches full email body via `resend.emails.receiving.get(email_id)`
4. Inserts row into `data_requests`
5. Sends acknowledgment email to sender
6. Creates `admin_notifications` for all allowlisted admins
7. Sends email alert to all allowlisted admins

**Important:** The webhook URL must use `https://www.wav.cash/api/email/inbound` (with `www`). Vercel redirects `wav.cash` to `www.wav.cash` with a 307 at the infrastructure level before middleware runs, which breaks webhook delivery.

### `GET /api/admin/data-requests`
Returns all data requests ordered by `received_at` desc. Requires admin auth.

### `PATCH /api/admin/data-requests`
Updates a request's fields. Body: `{ id, status?, request_type?, sender_name? }`. Requires admin auth.

### `POST /api/admin/data-requests/[id]/complete`
Marks a request as completed and sends the completion email. Body: `{ completion_summary, request_type?, sender_name? }`. Requires admin auth.

### `GET /api/admin/allowlist`
Returns all whitelisted admin emails. Requires admin auth.

### `POST /api/admin/allowlist`
Adds an email to the allowlist. Body: `{ email }`. Requires admin auth.

### `DELETE /api/admin/allowlist`
Removes an email from the allowlist. Body: `{ email }`. Self-removal and removal of the primary admin (`ibukun.oyewumij@gmail.com`) are blocked. Requires admin auth.

### `GET /api/admin/notifications`
Returns admin notifications for the authenticated admin, ordered by `created_at` desc. Requires admin auth.

### `PATCH /api/admin/notifications`
Marks notifications as read. Body: `{ ids?: string[] }` (specific IDs) or `{ all: true }` (mark all). Requires admin auth.

### `GET /api/admin/notifications/count`
Returns `{ count }` of unread admin notifications for the authenticated admin. Polled every 15s by the admin layout. Requires admin auth.

### `GET /api/admin/distributions/lookup?isrc=XXXX`
Looks up an active split by ISRC code. Returns track info, split details with contributors, activity events (`split_events`), and distribution history (`distributions` — filtered to successful, sorted desc). Requires admin auth.

### `GET /api/admin/distributions/balance?contract=0x...`
Checks distributable balances on a split contract. Returns AVAX balance + all supported token balances (USDC, EURC). Uses `hasDistributableBalance()` and `hasDistributableTokenBalance()` from `src/lib/contracts/interact.ts`. Requires admin auth.

### `POST /api/admin/distributions/[splitId]`
Admin-triggered distribution. Same logic as user-facing `POST /api/splits/[id]/distribute` but uses `verifyAdmin()` with no ownership check — admins can distribute any active split. Handles AVAX + ERC-20, DB inserts to `distributions`, contributor notifications, fee collection. Requires admin auth.

## Admin Dashboard Pages

### Data Requests (`/dashboard/admin/data-requests`)

- **Summary cards:** Count of received, in progress, and completed requests
- **Request list:** Expandable rows showing reference, sender, status badge, date
- **Expanded view:** Original email subject/body, editable sender name, request type dropdown
- **Status actions:** "Mark In Progress" button for received requests
- **Completion flow:** Textarea for completion summary, "Send Completion Email" button
- **Completed view:** Shows completion summary and completion date
- **Access denied:** Non-admin users see a clear "Access Denied" screen

#### Status Flow
```
received -> in_progress -> completed
```

Status badges: received (amber), in_progress (blue), completed (green).

### Distributions (`/dashboard/admin/distributions`)

Manual failsafe for distributing split payments when the cron job fails or for ad-hoc distribution.

1. **ISRC Lookup:** Enter an ISRC code, search returns the track and its active split
2. **Track Info Card:** Track title, ISRC, split title, contract address
3. **Contributors Table:** Name, role, share %, wallet address (truncated)
4. **Activity Timeline:** Unified chronological timeline merging split events (deployed, signed, activated, voided) and past distributions (payment amounts + token type). Same UI as the user-facing split detail page — vertical dot connectors, colored event badges, dates, and "View Record" explorer links for on-chain transactions
5. **Contract Balances:** Live AVAX + USDC + EURC balances from the contract (auto-fetched on lookup)
6. **Distribute Button:** Triggers distribution for all available balances. Shows result with amounts and tx hashes
7. **Balances auto-refresh** after a successful distribution

#### Activity Timeline Event Types

| Badge | Color | Description |
|-------|-------|-------------|
| DEPLOYED | Blue | Contract deployed on-chain |
| SIGNED | Amber | Contributor signed the agreement |
| ACTIVATED | Green | Agreement activated (all contributors signed) |
| VOIDED | Red | Agreement voided |
| PAYMENT | Green | Funds distributed (shows amount + token) |

Each event with a `tx_hash` links to Snowtrace (mainnet or testnet based on `NEXT_PUBLIC_CHAIN_ID`).

### Settings (`/dashboard/admin/settings`)

- **Admin Allowlist:** Add/remove admin emails
- Protected admin (`ibukun.oyewumij@gmail.com`) has no remove button and cannot be removed via the API
- Self-removal is also blocked
- Add email via input + "Add" button
- Remove email via X button (with loading state)

## DNS and Hosting

- **Domain:** `admin.wav.cash` — CNAME to `cname.vercel-dns.com`
- **Vercel:** Added as custom domain on the same Vercel project as `wav.cash`
- **Resend domain:** `noreply.wav.cash` (shared with invite emails)
- **Resend inbound:** MX records on `noreply.wav.cash` for receiving emails
- **Resend webhook URL:** `https://www.wav.cash/api/email/inbound` (must include `www`)
- **Porkbun forwarding:** `privacy@wav.cash` -> `privacy@noreply.wav.cash`

**Important:** The `send.noreply.wav.cash` MX record (pointing to `feedback-smtp.eu-west-1.amazonses.com`) is critical for SES bounce handling and must never be modified.
