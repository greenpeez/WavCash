# Admin Dashboard
> Internal admin panel for managing data rights requests, accessible only via `admin.wav.cash`.

## Overview

The Admin Dashboard handles privacy and data rights requests (GDPR, CCPA, etc.) submitted by users via email. It provides a single interface to view incoming requests, track their status, and send completion emails.

**Access:** `admin.wav.cash` only. The middleware redirects `admin.wav.cash/` to the data requests dashboard and blocks `/dashboard/admin/*` on any other hostname.

**Auth:** Privy authentication + email allowlist. Admin email is fetched directly from Privy (not the `users` table), so admins don't need a user-facing account. Only emails in the `admin_allowlist` table can access admin API routes. Non-admins see an "Access Denied" screen.

---

## Email Flow

```
User emails privacy@wav.cash
  -> Porkbun MX (fwd1/fwd2.porkbun.com)
  -> Porkbun forwards to privacy@noreply.wav.cash
  -> Resend inbound MX on noreply.wav.cash
  -> Resend fires webhook -> POST /api/email/inbound
  -> Webhook logs request to DB + sends acknowledgment email
```

### Acknowledgment Email (automatic)
- **From:** `WavCash Privacy <privacy@noreply.wav.cash>`
- **Subject:** "We received your data rights request — WavCash"
- **Content:** Confirms receipt, provides reference number (DR-XXXX), states 30-day completion window

### Completion Email (triggered from admin)
- **From:** `WavCash Privacy <privacy@noreply.wav.cash>`
- **Subject:** "Your data rights request has been completed — WavCash"
- **Content:** Summarizes what was done, includes supervisory authority links (CNPD, ICO, CPPA)

## Database

### `data_requests` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID (PK) | Auto-generated |
| `reference` | TEXT (unique) | Auto-generated `DR-0001`, `DR-0002`, etc. |
| `sender_email` | TEXT | From address of the incoming email |
| `sender_name` | TEXT | Extracted from email "From" field or manually set |
| `request_type` | TEXT | `access`, `deletion`, `correction`, `portability`, `opt-out`, `object`, `data_rights` |
| `status` | TEXT | `received` -> `in_progress` -> `completed` |
| `received_at` | TIMESTAMPTZ | When the email arrived |
| `completed_at` | TIMESTAMPTZ | When the completion email was sent |
| `completion_summary` | TEXT | Free text describing what was done |
| `original_subject` | TEXT | Subject line of incoming email |
| `original_body` | TEXT | Body text of incoming email |
| `resend_email_id` | TEXT | Resend's email ID for the inbound email |
| `created_at` | TIMESTAMPTZ | Row creation timestamp |

Reference numbers are auto-generated by a `BEFORE INSERT` trigger (`set_data_request_reference`).

### `admin_allowlist` table

| Column | Type | Description |
|--------|------|-------------|
| `email` | TEXT (PK) | Whitelisted admin email address |
| `created_at` | TIMESTAMPTZ | When the email was added |

Managed from the admin dashboard UI. Admins can add or remove emails. Self-removal is blocked. Both tables have RLS enabled with no policies — all access goes through the service client.

## API Routes

### `POST /api/email/inbound`
Resend webhook endpoint. No auth required (verified via webhook signature).

1. Verifies `webhook-id`, `webhook-timestamp`, `webhook-signature` headers against `RESEND_WEBHOOK_SECRET`
2. Filters for `email.received` events to `privacy@noreply.wav.cash`
3. Fetches full email body via `resend.emails.receiving.get(email_id)`
4. Inserts row into `data_requests`
5. Sends acknowledgment email to sender

### `GET /api/admin/data-requests`
Returns all data requests ordered by `received_at` desc. Requires admin auth.

### `PATCH /api/admin/data-requests`
Updates a request's fields. Body: `{ id, status?, request_type?, sender_name? }`. Requires admin auth.

### `POST /api/admin/data-requests/[id]/complete`
Marks a request as completed and sends the completion email. Body: `{ completion_summary, request_type?, sender_name? }`. Requires admin auth.

### `GET /api/admin/allowlist`
Returns all whitelisted admin emails. Requires admin auth.

### `POST /api/admin/allowlist`
Adds an email to the allowlist. Body: `{ email }`. Requires admin auth.

### `DELETE /api/admin/allowlist`
Removes an email from the allowlist. Body: `{ email }`. Self-removal is blocked. Requires admin auth.

## Admin Dashboard UI

**Route:** `/dashboard/admin/data-requests`

### Features
- **Summary cards:** Count of received, in progress, and completed requests
- **Request list:** Expandable rows showing reference, sender, status badge, date
- **Expanded view:** Original email subject/body, editable sender name, request type dropdown
- **Status actions:** "Mark In Progress" button for received requests
- **Completion flow:** Textarea for completion summary, "Send Completion Email" button
- **Completed view:** Shows completion summary and completion date
- **Admin allowlist:** Add/remove admin emails directly from the dashboard. Self-removal blocked.
- **Access denied:** Non-admin users see a clear "Access Denied" screen

### Status Flow
```
received -> in_progress -> completed
```

Status badges: received (amber), in_progress (blue), completed (green).

## DNS and Hosting

- **Domain:** `admin.wav.cash` — CNAME to `cname.vercel-dns.com`
- **Vercel:** Added as custom domain on the same Vercel project as `wav.cash`
- **Resend domain:** `noreply.wav.cash` (shared with invite emails)
- **Resend inbound:** MX records on `noreply.wav.cash` for receiving emails
- **Porkbun forwarding:** `privacy@wav.cash` -> `privacy@noreply.wav.cash`

**Important:** The `send.noreply.wav.cash` MX record (pointing to `feedback-smtp.eu-west-1.amazonses.com`) is critical for SES bounce handling and must never be modified.
